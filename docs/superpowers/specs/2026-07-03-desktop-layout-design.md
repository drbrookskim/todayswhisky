# 데스크톱 레이아웃 지원 설계

## 배경
현재 `index.html`은 모바일 전용으로 구현되어 있다. `.app-container`가 `max-width: 430px`로 고정되어 화면 중앙에 "스마트폰 목업" 형태로 떠 있고, 하단 탭바(홈/큐레이션/프로필)로 내비게이션한다. 이 앱을 데스크톱 브라우저에서도 자연스러운 웹서비스처럼 보이도록 확장한다.

## 목표
- 데스크톱(≥1024px)에서 웹서비스다운 넓은 레이아웃 제공
- 모바일 구현(로직/상태/시각)은 회귀 없이 100% 유지
- 하나의 `index.html` 안에서 반응형으로 전환 (별도 파일 분리 없음)

## 범위 밖
- React 컴포넌트 로직, MBTI/바이오리듬 분석 알고리즘, localStorage 스키마 변경 없음
- 새로운 기능/탭 추가 없음 — 기존 3탭(홈/큐레이션/프로필) 그대로
- 태블릿(768~1023px) 전용 별도 레이아웃은 설계하지 않음 — 이 폭에서는 모바일 레이아웃을 그대로 사용

## 브레이크포인트
Tailwind 기본 `lg:` (min-width: 1024px)를 데스크톱 전환 기준으로 사용한다. 1024px 미만은 기존 모바일 레이아웃을 그대로 사용한다.

## 네비게이션
- **모바일 (<1024px)**: 기존 `.bottom-nav` 하단 탭바 유지, 변경 없음.
- **데스크톱 (≥1024px)**: 상단 가로 네비바로 전환.
  - 좌측: 로고/타이틀("오늘의 위스키")
  - 중앙: 탭 메뉴 (홈 / 큐레이션 / 프로필) — 가로 배열, 활성 탭 하이라이트는 기존 `.nav-item.active` 색상(#14b8a6) 재사용
  - 우측: 언어 토글, 프로필 아이콘 등 기존 우상단 요소 재배치
  - `.bottom-nav`는 `lg:` 이상에서 `display: none`

## 콘텐츠 영역
- **모바일**: `.app-container`(430px 고정폭 + 박스섀도우) 그대로 유지.
- **데스크톱**: `.app-container`의 고정폭/박스섀도우("폰처럼 보이는" 스타일)를 `lg:` 오버라이드로 해제.
  - 콘텐츠는 `max-width: 1080px` 내외로 중앙 정렬
  - 좌우 여백에는 기존 배경 라디얼 그라디언트를 확장해 장식적 여백으로 채움 (새로운 배경 이미지/일러스트 추가 없이 CSS만으로 처리)
  - `.content-area`의 좌우 패딩을 데스크톱에서 더 넉넉하게 조정

## 탭별 재배치
- **HomeTab / CurateTab**: 카드 목록을 데스크톱에서 그리드로 재배치 (`grid-cols-1` → `lg:grid-cols-2` 또는 `lg:grid-cols-3`, 콘텐츠 성격에 따라 각 탭에서 적절히 선택). 카드 자체의 내부 마크업/로직은 변경하지 않고 감싸는 컨테이너의 className만 조정.
- **ProfileTab**: 필요 시 2열 배치(프로필 정보 + 설정/기록) 검토. 기존 세로 스택 구조를 유지하되 `lg:` 그리드로 좌우 분리 가능하면 분리, 무리하게 재배치하지 않는다.

## 온보딩/퀴즈/스플래시
`Splash`, `Onboarding`, `Quiz`, `DirectSelect` 화면은 데스크톱에서도 지금과 동일하게 좁은 카드형(모바일과 유사한 폭)을 유지한다. 데스크톱에서는 이 카드가 넓은 배경 중앙에 뜨는 형태가 되며, 별도의 데스크톱 전용 레이아웃을 만들지 않는다. 기존 JSX/클래스 대부분 재사용.

## 구현 방식
- 순수 CSS/Tailwind 유틸리티 클래스 변경으로 구현 (`lg:` prefix 활용)
- `<style>` 블록에 `.desktop-nav`, `.app-container`의 `lg:` 오버라이드 등 필요한 CSS 규칙 추가
- 컴포넌트 변경은 `App`(상단 네비 vs 하단 네비 분기 렌더링), `HomeTab`, `CurateTab`, `ProfileTab`의 className 조정에 한정
- 상태 관리(useState 등), 데이터/텍스트 정의(`TEXT`), 분석 로직 함수는 전혀 건드리지 않음

## 회귀 방지
- 1024px 미만 화면에서는 기존 클래스/스타일이 그대로 적용되도록, 모든 신규 스타일은 `lg:` 이상에서만 활성화되게 작성한다.
- 기존 실기기(모바일) 스크린샷과 비교해 시각적 차이가 없는지 확인한다.

## 테스트 계획
- 브라우저 프리뷰에서 뷰포트를 모바일(375px), 태블릿(768px), 데스크톱(1280px, 1440px)으로 리사이즈하며 확인
  - 모바일/태블릿: 기존과 동일한 하단 탭바 + 430px 카드 레이아웃 유지 확인
  - 데스크톱: 상단 네비바 노출, 콘텐츠 그리드 재배치, 온보딩/퀴즈 카드형 유지 확인
- 각 탭(홈/큐레이션/프로필) 전환 및 온보딩→퀴즈→결과 플로우를 데스크톱 폭에서 한 번씩 실행해 기능 회귀 없는지 확인
